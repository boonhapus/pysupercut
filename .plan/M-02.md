# M-02 — Video hamming sliding window

## Goal
Given two files' `FrameHash` lists, find contiguous matching frame sequences.

## Algorithm
1. Slide a window of B's hashes over A's hashes
2. At each alignment, compute `hamming(a_hash, b_hash)` per frame pair
3. Find longest run of consecutive pairs where hamming distance < `--phash-threshold`
4. Run must be ≥ `--min-overlap * fps` frames long

## Output
```python
@define
class VideoMatch:
    range_in_a: tuple[float, float]
    range_in_b: tuple[float, float]
    score: float    # 1 - mean(hamming_distances / 64)
```

## Notes
- Hamming distance on 64-bit hash: 0 = identical, 64 = opposite. Threshold of 10 = very similar.
- Use numpy for the hamming computation — XOR + popcount over arrays.
- If no run found → return `None`.

## Done when
Given two frame hash lists from files with known 30s overlap at 1fps, returns `VideoMatch` covering that range (±2 frames tolerance).
