# ST-01 — Re-encode trimmed segments to temp files

## Goal
Each `Segment` with `needs_reencode=True` is cut frame-accurately and written to a temp file.

## Steps
1. Create a `tempfile.TemporaryDirectory` — hold reference for cleanup in ST-04
2. For each segment where `needs_reencode=True`:
   ```
   ffmpeg -i <source> -ss <start> -to <end> -c:v libx264 -c:a aac <tempdir>/<stem>_trimmed.mp4
   ```
3. Run all re-encodes concurrently: `asyncio.gather(*[reencode(seg) for seg in trimmed_segs])`
4. On non-zero exit → log stderr + hard error with which file failed
5. Return map: `{segment → temp_path}`

## Notes
- `-ss` before `-i` = fast seek but less accurate. `-ss` after `-i` = slow but frame-accurate. Use `-ss` AFTER `-i` here since frame accuracy is required (user chose this tradeoff in planning).
- `-c:v libx264 -c:a aac` produces universally compatible output safe for concat.
- Temp files named deterministically: `<original_stem>_<int(start)>_<int(end)>.mp4`

## Done when
A segment with start=30, end=90 produces a temp file of duration ≈ 60s. Verify with ffprobe.
