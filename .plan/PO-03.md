# PO-03 — Handle files with no audio stream

## Goal
Files without audio don't crash the tool — they fall back to video-only matching with a confidence penalty.

## Steps
1. In F-01: if `VideoFile.has_audio == False` → return empty `[]`, log warning
2. In M-03: if `audio_chunks` is empty for either file in a pair:
   - Skip audio match entirely
   - Use video match alone if score > `--phash-threshold`
   - Cap confidence at 0.6 (video-only, less reliable)
   - Log: `"Audio unavailable for <file> — using video-only match (confidence capped at 0.6)"`
3. Emit `OverlapMatch` with capped confidence if video match found

## Notes
- This means a video-only match CAN produce an overlap — but only if video hamming score is strong.
- User may want to tune `--phash-threshold` lower when working with audio-less files.

## Done when
A file with no audio stream (e.g. screen recording) produces a valid `OverlapMatch` based on video alone.
A file with no audio and weak video match produces no overlap.
