# PO-04 — Handle complex interleave edge case

## Goal
Don't crash when overlaps produce an unresolvable timeline (e.g. cyclic or deeply tangled graph).

## What this looks like
- A overlaps B at tail, B overlaps A at tail (circular)
- After trimming, a segment has negative duration
- Overlap graph has no valid linear ordering

## Steps
1. After T-03, validate every segment: `keep_start < keep_end`
2. If any segment has `keep_start >= keep_end` and wasn't already DROPPED → log warning + mark DROPPED
3. After all segments resolved, verify at least 1 segment remains → hard error if list is empty
4. Log a summary of any segments dropped due to timeline conflict (distinct from containment drops)

## Notes
- Don't attempt to "solve" complex graph cases — just detect and drop conflicted segments conservatively.
- This is an edge case from adversarial or unusual input. Log clearly so user can investigate.

## Done when
A pathological input (A and B mutually overlap each other's full duration) produces a clear log and drops the conflicted file rather than crashing.
