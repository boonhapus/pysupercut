# T-02 — Keep-range computation → trim B's start

## Goal
For each overlap between consecutive files (A before B in known order), trim B's start by the overlap duration. A is never trimmed.

## Rule
- Known order = user-supplied file order
- For overlap (A, B) where A precedes B: B's keep-range start = `range_b[1]` (end of overlap in B)
- A's keep-range is never modified by this rule

## Steps
1. For each file, initialise keep-range = `(0.0, duration)`
2. Sort `OverlapMatch` list by position of `file_a` in known order
3. For each match where A precedes B: set `B.keep_start = max(B.keep_start, range_b[1])`
4. Keep-range = `(keep_start, duration)`

## Notes
- `max()` handles the case where multiple overlaps push B's start further forward.
- Non-adjacent overlaps (A overlaps C) handled here too — same rule applies regardless of adjacency.

## Done when
Given overlap of 30s between A and B, B's keep-range starts at 30s.
A's keep-range is unchanged (0 → duration).
