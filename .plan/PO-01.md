# PO-01 — structlog throughout all phases

## Goal
All significant events logged with structured context, not print statements.

## Events to log (minimum)
| Phase | Event | Level |
|-------|-------|-------|
| Probe | file probed | DEBUG |
| Probe | codec mismatch | ERROR |
| Fingerprint | file fingerprinted | DEBUG |
| Fingerprint | no audio stream | WARNING |
| Match | overlap found | INFO |
| Match | signals disagree | WARNING |
| Timeline | file dropped | INFO |
| Timeline | segment keep-range | DEBUG |
| Stitch | re-encoding segment | INFO |
| Stitch | concat complete | INFO |
| Stitch | duration mismatch | WARNING |

## Setup
```python
import structlog
structlog.configure(
    processors=[
        structlog.processors.TimeStamper(fmt="iso"),
        structlog.processors.add_log_level,
        structlog.dev.ConsoleRenderer(),   # human-readable in terminal
    ]
)
log = structlog.get_logger()
```

## Done when
Running the tool on a test set produces clean structured log output for all phases. No bare `print()` calls remain (except `--dry-run` report and final summary).
