# M-01 — Audio XOR sliding window

## Goal
Given two files' `AudioChunk` lists, find the time offset where their audio most closely matches.

## Algorithm
1. For each offset `o` (shifting B's chunks relative to A's):
   - Find chunk pairs that align at offset `o`
   - XOR their fingerprint arrays element-wise → count matching bits
   - Score = matching bits / total bits
2. Return the offset and time range where score exceeds `--audio-threshold`

## Output
```python
@define
class AudioMatch:
    offset_secs: float
    range_in_a: tuple[float, float]
    range_in_b: tuple[float, float]
    score: float
```

## Notes
- Only test offsets where chunks actually overlap in time — don't brute-force all combinations.
- If no offset exceeds threshold → return `None`.
- Chunk size from F-01 sets resolution. 5s chunks → 5s accuracy on overlap boundary. Acceptable for MVP.

## Done when
Given two audio chunk lists from files with known 30s overlap, returns an `AudioMatch` with correct range (±5s tolerance given chunk size).
