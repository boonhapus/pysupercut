# F-04 — Validate fingerprint output completeness

## Goal
Hard error if any file produced no usable fingerprint data before match phase begins.

## Steps
1. After F-03, for each file check:
   - `has_audio == True` → must have `len(chunks) >= 1`
   - Always → must have `len(hashes) >= 1`
2. If either fails → log which file + which signal is empty + hard error

## Done when
A file with corrupted video stream or 0-duration causes a clear error before the match phase runs.
