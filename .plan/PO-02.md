# PO-02 — Progress reporting per phase

## Goal
User sees what the tool is doing during long-running operations (large files, many pairs).

## Approach
Simple print-based progress — no external library needed.

```
[1/3] Probing files...         ✓ 3 files probed
[2/3] Fingerprinting...        ✓ a.mp4  ✓ b.mp4  ✓ c.mp4
[3/3] Matching pairs (3)...    ✓ (a,b)  ✗ (a,c)  ✓ (b,c)
```

## Steps
1. Print phase header before each phase starts
2. Print per-file/per-pair tick as each completes (use callback from asyncio.gather results)
3. Print phase summary on completion

## Notes
- Keep it simple — a few `print()` calls is fine here. No rich/tqdm needed for MVP.
- Don't interleave with structlog output — phase prints go to stdout, structlog to stderr, or configure structlog to suppress DEBUG during normal runs.

## Done when
Running on 3 files shows clear per-phase progress without garbled output.
