# T-03 — Double-sided trim (B overlaps both A and C)

## Goal
Handle the case where file B has overlapping content at both its head (with A) and its tail (with C).

## Logic
After T-02, B's `keep_start` may already be pushed forward.
Now also check: for overlaps where B is `file_a` and C is `file_b` (B precedes C):
- B's keep-range end = `min(B.keep_end, range_a_end)` where `range_a` is B's range in the (B, C) match

```
B keep-range = (keep_start_from_T02, overlap_with_C_start_in_B)
```

## Steps
1. For each match where B precedes C: set `B.keep_end = min(B.keep_end, range_a[0])`  
   (`range_a[0]` = where B's overlap with C begins in B's timeline)
2. After both passes: if `keep_start >= keep_end` → mark B as DROPPED (no unique content remains)
3. Log: `"Dropping <file> — no unique content after trimming both ends"`

## Done when
B with 10s head overlap (with A) and 10s tail overlap (with C) and total duration 15s → keep-range = (10, 5) → `keep_start >= keep_end` → DROPPED.
B with 60s total, 10s head + 10s tail overlap → keep-range = (10, 50) → kept, 40s of unique content.
